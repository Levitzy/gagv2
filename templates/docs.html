GET

/api/stocks/{category}

Retrieve stock data for a specific category only.

Request

GET {{ request.url_root }}api/stocks/seed

Parameters

category

The stock category to retrieve

Required
Valid values: seed, gear, egg, cosmetic, honey, easter, night

Response Example

{
  "seed": [
    {
      "name": "Carrot",
      "value": 15
    },
    {
      "name": "Strawberry",
      "value": 3
    }
  ]
}

Error Response (Invalid Category)

{
  "error": "Invalid category. Valid categories are: seed, gear, egg, cosmetic, honey, easter, night"
}
GET

/api/weather

Retrieve current weather conditions and upcoming weather changes.

Request

GET {{ request.url_root }}api/weather

Response Example

{
  "currentWeather": {
    "name": "Sunny",
    "description": "Clear skies and bright sunshine"
  },
  "nextWeatherTimestamp": 1641024000000,
  "nextWeatherTimestampISO": "2022-01-01T12:00:00",
  "nextWeatherFormatted": "jan 1 2022 - 12:00pm",
  "specialWeathers": {
    "rainEvent": {
      "timestamp": 1641110400000,
      "timestampISO": "2022-01-02T12:00:00",
      "timestampFormatted": "jan 2 2022 - 12:00pm"
    }
  }
}

Response Fields

Field Type Description
currentWeather Object Current weather conditions with name and description
nextWeatherTimestamp Number Unix timestamp in milliseconds for next weather change
nextWeatherFormatted String Human-readable timestamp for next weather change
specialWeathers Object Special weather events with timestamps

HTTP Status Codes

200 OK

Request successful, data returned successfully

400 Bad Request

Invalid category parameter provided

404 Not Found

Requested endpoint does not exist

500 Internal Server Error

Failed to scrape data from GrowAGarden.gg or server error

Usage Guidelines

Rate Limiting

No formal rate limits enforced, but please use responsibly to maintain service quality.

Caching

Data is fetched live from the source. Consider implementing your own caching for frequently accessed data.

Best Practices

Cache responses when possible to reduce load
Implement proper error handling for reliability
Use specific category endpoints when possible
Monitor for API changes and updates

Prohibited Usage

Do not spam the API, use it for malicious purposes, or create excessive load that could impact the source website.
